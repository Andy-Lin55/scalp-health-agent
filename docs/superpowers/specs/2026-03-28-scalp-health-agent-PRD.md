## 一、产品概述

### 1.1 产品定位

面向普通消费者的 AI 驱动头皮健康管理 Agent，帮助用户：
- 识别和理解自己的头皮问题与脱发状况
- 通过对话和图片分析获得专业级诊断参考
- 获取个性化、综合性的改善方案（产品、饮食、习惯、补剂、就医）
- 长期追踪头皮健康状态变化和护理效果

### 1.2 目标用户

普通消费者，尤其是：
- 有脱发、头屑、头皮痒等困扰但不确定严重程度的用户
- 用过多种产品但效果不佳、不知如何选择的用户
- 希望系统性管理头皮健康、追踪改善进度的用户

### 1.3 核心价值

> "像有一位专业头皮顾问随时陪伴，从诊断到方案到长期追踪，一次对话搞定"

## 三、系统架构

### 3.1 架构选型：方案 B — 带工具调用的模块化 Agent

一个主对话 Orchestrator Agent，负责：
- 理解用户意图（多轮对话上下文）
- 按需调用专属工具模块
- 整合各工具结果，以自然对话形式回复用户

用户感知：自然的对话体验
后台实现：结构化工具调用

### 3.2 AI 模型路由

| 用途 | 主模型 | 备用/补充 |
|------|--------|-----------|
| 对话推理 | Claude API | 国内大模型（合规备用：通义千问/文心一言） |
| 图片分析 | GPT-4V / 专业视觉模型 | 自建视觉模型（长期） |
| 产品匹配 | Embedding 向量相似度 | — |

### 3.3 工具模块总览

| # | 工具名 | 英文标识 | 核心职责 |
|---|--------|----------|----------|
| 1 | 头皮诊断工具 | `ScalpDiagnosisTool` | 标准化症状收集、问题分类、诊断输出 |
| 2 | 图片分析工具 | `PhotoAnalysisTool` | 17项指标视觉分析、产品照片识别 |
| 3 | 方案推荐工具 | `SolutionRecommendationTool` | 5类综合改善建议 |
| 4 | 健康档案工具 | `HealthProfileTool` | 档案管理、进展追踪、完整报告生成 |

## 四、跨工具记忆系统

所有工具共享统一记忆架构，实现跨会话连续性。

### 4.1 短期记忆（Session Memory）

- 当前会话对话上下文
- 本次对话中收集的新信息
- 临时状态（等待上传、已跳过、待确认等）
- 当前轮次意图追踪

### 4.2 长期记忆（Persistent Memory）

- 结构化健康档案（详见工具 4）
- 历次诊断结果时间线
- 历次图片分析结果（含17项指标历史值）
- 产品使用记录与成分档案
- 方案执行记录与效果反馈

### 4.3 记忆召回（Memory Recall）

- 对话开始时自动加载档案摘要注入上下文
- Agent 主动引用历史记录（"上次你提到…"）
- 用户可直接询问历史（"我上次的诊断结果是什么？"）
- 关键信息跨会话串联，不要求用户重复说明

### 4.4 自动更新触发

| 触发事件 | 写入内容 |
|----------|----------|
| 完成一次诊断 | 诊断结果 + 当前头皮状态更新 |
| 完成一次图片分析 | 17项指标评分 + 综合健康评分更新 |
| 识别到产品照片 | 产品名称 + 成分表 + 功效信息 |
| 对话中获取新字段 | 增量更新对应档案字段 |

## 五、用户体验设计

### 5.1 首次用户流程

```
Agent 问候 → 引导式对话建档 → 信息逐步完善 → 个性化方案输出
```

**关键原则：**
- 问卷字段 = 对话问题，用户无感建档
- 照片邀请以自然气泡形式嵌入对话，可忽略继续聊
- 快捷选项气泡（点选或文字均可）降低输入门槛
- 随时可给基础建议，信息越多越精准

### 5.2 对话示例（首次）

```
Agent: 你好！我是你的头皮健康顾问。可以告诉我你最近的主要困扰吗？

用户: 最近掉发比较多，洗头的时候特别明显

Agent: 大概持续多久了？另外，头皮平时是偏油还是偏干？
      [偏油] [偏干] [中性] [不确定]

用户: 大概三个月了，头皮偏油

Agent: 了解了。油性头皮配合脱发三个月，有几种可能原因需要排查。
      📷 拍一张发际线或头顶的照片，我能更准确判断脱发类型。
         直接发过来就好（也可以继续文字描述）
```

### 5.3 回访用户体验

- 自动加载历史档案，续接上次状态
- Agent 主动问近况，引用上次结论
- 对比历史照片或指标数据，评估改善情况
- 动态调整推荐方案

## 六、工具详细需求

### 6.1 头皮诊断工具（ScalpDiagnosisTool）

#### 触发条件
- 用户首次描述头皮困扰
- 用户主动提出诊断需求（"帮我看看"、"分析一下"、"做个检测"）
- 用户描述模糊、需要标准化
- 回访时症状发生变化
- 图片分析完成后联合触发

#### 引导式标准化流程

```
① 头皮类型评估
   → 油性 / 干性 / 混合 / 敏感 / 正常

② 问题分类
   → 脱发类型：雄激素性 / 休止期 / 斑秃 / 其他
   → 头皮问题：头屑 / 炎症 / 瘙痒 / 油腻 / 其他

③ 症状描述标准化
   → 程度（轻/中/重）
   → 频率与持续时长
   → 诱因识别

④ 诊断输出 + 档案写入 + 鼓励上传照片
```

#### 采集字段（通过对话逐步收集，不重复询问已知字段）

主诉症状、持续时长、头皮类型、脱发区域、伴随症状（痒/屑/红/异味）、洗发频率、压力水平、饮食习惯、睡眠情况、家族脱发史、在用产品、近期药物/激素变化

#### 输出内容

- 头皮类型评估结果
- 问题分类与严重程度（轻度 / 中度 / 需就医）
- 主要诱因分析
- 置信度说明（与信息完整度挂钩）
- 严重情况明确提示就医

#### 档案联动

每次诊断结果自动增量写入健康档案时间线，不覆盖历史记录，同时更新当前状态字段。

### 6.2 图片分析工具（PhotoAnalysisTool）

#### 触发条件

- 用户主动上传照片
- 对话建档到一定阶段，Agent 发出自然邀请
- 回访时对比历史照片

#### 云端分析引导逻辑

用户上传照片时，Agent 提示：
> "本地分析准确度有限，建议授权云端模型分析，效果更精准。如不方便，也可继续本地模式。"

用户明确同意后方可上传至服务器。

#### 拍摄引导

Agent 提供拍摄示范说明：
- 发际线正面照
- 头顶 / 分缝处俯视照
- 头皮特写（有明显症状处）

#### 分析指标体系（17 项）

**① 头皮物理性状**
- 头皮敏感及炎症（红/脓/肿）
- 角质层状态（头屑类型/厚度）
- 头皮弹性

**② 头皮分泌物**
- 油脂分泌程度
- 毛囊堵塞情况

**③ 毛囊与毛发情况**
- 毛囊健康度
- 毛发直径
- 头发密度
- 头皮菌群迹象（痤疮皮肤杆菌 / 表皮葡萄球菌 / 马拉色菌等）

**④ 头皮代谢**
- 头皮毛细血管（血液微循环）
- 头皮水分状态

**⑤ 脱发等级**
- 发际线后退程度
- 脱发部位分布
- Hamilton-Norwood / Ludwig 分级参考

#### 图片质量不足处理

若图片不清晰，提示用户图片质量不足以完成分析，鼓励使用或购买专业头皮检测设备（皮肤镜/头皮检测仪），并提供推荐设备类型。

#### 产品照片自动识别

用户发送正在使用的产品照片时：
1. 自动识别品牌和产品名
2. 联网 / API 查询成分表和功效信息
3. 保存至健康档案产品库

#### 档案联动

每次分析结果（含17项指标评分）自动写入健康档案，保留历史数据用于趋势对比，同时更新综合健康评分。

#### 隐私处理

- 默认本地分析，不上传服务器
- 云端分析须用户明确授权
- 匿名脱敏后可选贡献训练数据
- 用户可随时删除所有照片记录

### 6.3 方案推荐工具（SolutionRecommendationTool）

#### 前置步骤：了解现用产品

在给出推荐前，Agent 主动引导：
> "你现在在用什么洗发水或护发产品？可以直接描述，或者拍个照片发给我。"

目的：结合现有方案给出调整建议，避免重复推荐或成分冲突。

#### 推荐优先级逻辑

```
严重情况（炎症/感染/大量脱发）→ 就医建议优先
中轻度 → 产品 + 生活方案并列输出
信息不足 → 给出基础通用建议并说明局限性
```

#### 五类推荐内容

**1. 产品推荐**
- 自有产品库优先匹配，提供更便捷的购买入口和客户体验
- 开放平台补充：京东 / 淘宝 / 小红书 API
- 按成分、功效、价位匹配，Embedding 向量相似度排序

**2. 饮食营养建议**
- 针对头发生长的关键营养素：铁、锌、生物素、蛋白质、Omega-3、维生素 D/E
- 结合用户饮食习惯，给出具体可行的调整建议

**3. 生活习惯调整**
- 睡眠优化、压力管理
- 洗护频率和手法指导
- 运动/血液循环改善
- 吹风/造型习惯调整

**4. 成分/补剂指引**
- 口服补剂建议（生物素、锌、胶原蛋白等）
- 外用成分推荐（米诺地尔、咖啡因、锯棕榈等）
- 注意事项与禁忌说明
- 明确声明：仅供参考，不作处方建议

**5. 就医建议**

触发条件：
- 疑似感染/炎症
- 短期大量脱发
- 伴随全身症状
- Agent 诊断置信度低

输出内容：推荐科室（皮肤科/毛发专科）、就医前准备建议

### 6.4 健康档案工具（HealthProfileTool）

#### 档案结构

| 字段类别 | 具体内容 |
|----------|----------|
| 基础信息 | 年龄、性别、体质、联系方式（可选） |
| 头皮状态 | 当前头皮类型、当前主要问题 |
| 诊断时间线 | 历次诊断结果（时间戳、类型、严重度、诱因）|
| 图片分析历史 | 历次17项指标评分、照片（加密存储）|
| 问卷完成度 | 各字段填写状态，已知字段不重复询问 |
| 产品档案 | 在用产品、历史产品、成分表、使用效果 |
| 方案记录 | 执行中的护理方案、开始时间、执行情况 |
| 效果反馈 | 用户主观反馈 + 指标客观变化 |
| 综合健康评分 | 基于所有可用数据的综合评分（0-100）|

#### 数据存储策略

| 级别 | 默认 | 说明 |
|------|------|------|
| 本地加密存储 | ✅ 默认开启 | 设备端 AES 加密，不上传服务器 |
| 云端账号同步 | ⭕ 用户可选 | 登录账号后加密上传，支持多设备访问 |
| 匿名数据贡献 | ⭕ 授权后开启 | 脱敏加密后用于模型迭代，用户随时可撤回 |

#### 进展追踪功能

- 护理方案执行提醒（可选开启）
- 定期复查提示（2~4 周周期）
- 照片前后对比展示
- 17项指标历史趋势图
- 方案效果评估
- 档案完整度进度条（引导用户持续完善）

#### 完整报告生成

**触发方式：** 用户说出"完整报告"、"总结一下"、"生成报告"、"我想看全部情况"等表述时，Agent 自动识别意图并触发。

**报告内容：**
- 头皮健康总体评估
- 症状演变时间线
- 17项指标历史趋势
- 使用产品汇总与效果回顾
- 当前推荐方案

**输出格式：**
- 对话内摘要展示
- 可导出 PDF
- 可分享给医生（脱敏版）

## 七、数据与隐私设计

### 7.1 数据分层

```
用户设备（本地加密）
    ↕ 用户授权后同步
云端账号数据（加密存储）
    ↕ 用户授权后匿名贡献
模型训练数据池（脱敏 + 加密）
```

### 7.2 隐私原则

1. **最小化采集**：只收集诊断和推荐所必需的信息
2. **本地优先**：数据默认留在用户设备，不主动上传
3. **明确授权**：云端同步和数据贡献均需单独授权，可随时撤回
4. **透明说明**：用户随时可查看 Agent 掌握了哪些关于自己的信息
5. **不替代医疗**：所有诊断仅供参考，不构成医疗建议

## 八、产品推荐逻辑

### 8.1 自有产品优先策略

- 自有产品库与诊断结果进行向量匹配
- 自有产品在推荐列表中优先展示
- 为自有产品提供更便捷的购买入口和客户服务体验

### 8.2 开放平台接入

- 京东、淘宝、小红书等电商平台 API 集成
- 作为自有产品的补充，覆盖自有产品库未涵盖的品类

### 8.3 匹配算法

- 基于头皮问题类型、症状、诊断结果生成需求向量
- 产品按功效成分建立向量索引
- 余弦相似度排序，结合用户价位偏好过滤

## 九、非功能性需求

| 维度 | 要求 |
|------|------|
| 响应时间 | 对话回复 ≤ 3 秒，图片分析 ≤ 15 秒 |
| 合规性 | 支持国内大模型备用，满足数据本地化要求 |
| 免责声明 | 所有诊断结论附"仅供参考，不替代专业医疗诊断"说明 |
| 多语言 | 优先中文，架构预留多语言扩展 |
| 离线能力 | 本地存储档案可离线查看，分析功能需联网 |
